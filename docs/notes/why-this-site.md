# 为什么要有这个网站

我想要一个不依赖信息流的地方。文档、项目里的取舍，以及偶尔写成形的想法，都可以按自己的方式摆放，不必等平台替它们决定顺序和寿命。

独立网站也没有那么神秘：它只是一个能持续修改、随时查找，并且由自己决定结构的目录。今天写得不够准确，之后还可以回来改。

## 这里会放什么

- 实际用过，还会再次查阅的步骤
- 项目的说明、更新和复盘
- 不适合塞进信息流的长一点的文字

栏目不会为了显得完整而填满。内容准备好，再放进来。

## 怎么更新

正文用 Markdown 编写，和网站源码一起存放在 GitHub。推送到 `main` 后，GitHub Actions 会自动构建并发布。

流程很短，所以修正一处措辞也不必等到所谓的“大版本”。
